# KAI · 移动电商兴趣意图 Agent Demo

## 项目目标
让**业务负责人与评估人员**更好地理解用户在电商场景下的**行为与兴趣变化**，并判断推荐内容对用户**即时兴趣 / 长期兴趣**的**承接是否匹配**。

## 布局（三栏）
- **左栏 · 用户行为 / Session / Trace**：实时会话画像（时长、行为数、页面深度、曝光→详情→加购漏斗）、会话与轨迹列表、预设剧本自动演示、Agent 观察流。
- **中栏 · 移动电商 App 实机演示 / 回放**：手机内浏览/搜索/筛选/详情/加购；点击任一轨迹可「回放」复现当时的浏览与曝光情况，支持暂停、时间轴拖动跳转。
- **右栏 · 用户画像 + 推荐流打分**：用户画像（性别/消费力/平均笔单价/价位带/生活画像猜测/历史品类）、意图漏斗（浏览→对比→决策→下单）+ 置信趋势、当前用户状态、推荐流总评 + 运营策略开关、逐项打分（意图精准 / 兴趣延展 / 价格匹配）。

## 核心能力
- **行为采集**：hover/view/search/filter/sort/detail/fav/cart/dismiss/purchase，带 session 与 trace_id。
- **兴趣引擎**：短期（25s 半衰期）与长期（10min 半衰期）双时间尺度，支持偏好漂移指示。
- **意图推断**：广泛浏览 / 对比决策 / 强购买 / 价格敏感 四类意图 + 置信度 + 状态机漏斗。
- **冷启动 → 个性化**：行为不足 6 步用热销兜底，达到后切个性化推荐，思考流播报切换。
- **ε-greedy 探索**：个性化期以 ~22% 概率注入探索商品，标注「探索」。
- **负反馈**：卡片 ✕「不感兴趣」会降低该品类兴趣并移出推荐。
- **运营策略**：默认 / 秒杀加权 / 新品曝光 / 清库存，影响推荐打分。
- **回放归因**：复现浏览路径与曝光，统计曝光/互动/加购漏斗。

## 运行
浏览器直接打开 `index.html`（单文件，无需联网、无需依赖）。
